Types & literals

Built-in types and their literal syntax.

TypeLiteral / example

String interpolation

In double-quoted strings: "Hello {name}", "Sum: {total}". Format specs: {name:>10}, {x:.2f}.

Collections

TypeLiteral / example
list[1, 2, 3], []
dict{"a": 1, "b": 2}, {}
tuple(a, b), ()
setSet of strings (via built-in or constructor).
counterFrequency counter (via built-in).

Ranges

start..end (inclusive end), start..<end (exclusive). Optional step in some contexts. range(n), range(a, b), range(a, b, step).

Special literals

inf / -inf, #hex, date"...", time"...", datetime"...", $1.50 (money), emoji. _ (underscore) is the anonymous variable.

Type names (for declarations)

int float str bool list dict tuple vec mat set counter char date time datetime interval — and identifier for custom types (class, enum).

Indexing and slicing

list[i], list[start:end], list[start:end:step]. Same for strings. Dict: d[key], d.get(key).
