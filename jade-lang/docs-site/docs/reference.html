Language reference

Values, output, and code from basic to advanced — all on one page. Use your browser’s search to find a topic.

Values

Definition and Usage. Every value in Jade has a type. You can declare variables with an optional type (int: x = 42) or let the interpreter infer it (x = 42). Top-level declarations are order-independent.

Introduction

Jade is dynamically typed. Types include integers, floats, strings, booleans, characters, lists, dicts, tuples, ranges, and specials like inf, dates, and money.

Primitive types

TypeDescriptionExample literal
intSigned integer42, -1, 0
floatDouble-precision float3.14, 1e-2, -0.5
strUTF-8 string"hello", ""
boolBooleantrue, false
charSingle character'a', ' '

Primitive values

NameDescription
true and falseBoolean values
noneAbsence of a value (optional / no return)

String literals

Double-quoted: "hello". Interpolation: "Sum: {total}". Format specs: {x:.2f}, {name:>10} (see Formatting).

Escape sequences

In double-quoted strings:

EscapeMeaning
\nNewline
\tTab
\rCarriage return
\\ \" \'Backslash, double quote, single quote
\a \b \f \v \0 \? \eBell, backspace, form feed, vertical tab, null, ?, ESC
\xHHHex byte (2 hex digits)
\U{HHHHHH}Unicode code point (1–6 hex digits)
\c{name}ANSI: red, green, yellow, blue, magenta, cyan, white, bold, dim, ul, reset
\emoji{name}Emoji: fire, heart, sparkles, rocket, star, thumbs_up, smile, check, cross, warning, etc.

Assignment

Syntax

type: name = value   # With type
name = value         # Inferred

Example

list: xs = [1, 2, 3]
dict: d = { "a": 1, "b": 2 }

# Inferred (no type):
n = 42
name = "Jade"
xs = [1, 2, 3]

Output (out)

Definition and Usage. out() prints to stdout. It supports plain values, multiple arguments, ANSI colors and styles, tables, progress bars, animations, and gradient text. Use out(x) to print one value with a newline, or out(a, b, c) to print several space-separated, or out(text, options) for styled output.

Syntax

out(value)
out(a, b, c)
out(text, { "color": "red", "style": "bold", "progress": 50, "animate": "spinner", "gradient": ["#00ff00", "#ff0000"] })

Code

Formatting

First arg string with {}, second arg dict: {name} replaced by value. Specs: {name:>N} right-align, {name: left-align, {name:.2f} float with 2 decimals.

Code
str: user = "Jade"
int: count = 42
out("User: {user}, count: {count}", { "user": user, "count": count })
out("Right: {x:>10}", { "x": 123 })
out("Float: {pi:.2f}", { "pi": 3.14159 })
Output
User: Jade, count: 42
Right:        123
Float: 3.14

when

Definition and Usage. when matches a value against string (or other) cases. Use it for multi-way dispatch on a single value.

Syntax

when value { case "a" > expr; case "b" > expr }
Code
str: cmd = "start"
when cmd {
  case "start" > out("Starting...")
  case "stop" > out("Stopping...")
  case "pause" > out("Paused")
}
Output
Starting...

unless

Definition and Usage. unless runs a block only when the condition is false. Opposite of if.

Syntax

unless condition { statements }
Code
bool: done = false
unless done { out("Still working...") }
done = true
unless done { out("Done") }
out("Finished")
Output
Still working...
Finished

either

Definition and Usage. either matches a value with pipeline-style cases (|> value > expr). Handy for boolean or enum-like choices.

Syntax

either value { |> val1 > expr; |> val2 > expr }
Code
bool: ok = true
either ok {
  |> true  > out("Yes")
  |> false > out("No")
}
Output
Yes

while

Definition and Usage. while repeats a block while the condition is true. Use for countdowns or until a flag changes.

Syntax

while condition { statements }
Code
int: n = 3
while n > 0 {
  out("Count: " + n)
  n = n - 1
}
out("Go!")
Output
Count: 3
Count: 2
Count: 1
Go!
defer cleanup()
panic("fatal")
guard n >= 0 > return 0
return factorial(n - 1) * n

List methods (push, pop, insert, remove)

Definition and Usage. push(list, item) returns a new list with the item at the end. pop(list) returns the last element (the list is consumed). insert(list, index, item) returns a new list with the item at index. Use method style too: list.push(x), list.pop(), list.append(other_list).

a = push(a, 30)
out(a)
a = insert(a, 1, 15)
out(a)

out(last)

out(c)

Progress bar, animation, gradient

Definition and Usage. out(text, options) accepts progress (0–100), animate (e.g. "spinner", "dots"), and gradient (list of hex colors) for styled terminal output.

Example (progress)

out("Loading", { "progress": 75, "width": 40, "color": "green" })

Example (animation)

Example (gradient text)

In a terminal, these show a progress bar, a short spinner animation, and colored gradient text respectively.

Grammar summary

  • Function: fn [type?] | name ( params ) > body. Params: x or type | x.
  • Variable: type: name = expr.
  • Block: { stmt; stmt }. Newlines separate statements.
  • String interpolation: "text {var}", "{var:.2f}".

Operators (precedence high to low)

  • Unary: not - ~
  • Power: **
  • Factor: * / %
  • Term: + -
  • Shift: << >>
  • Bitwise: & ^ |
  • Comparison: < > <= >=
  • Equality: == != ~==
  • Logical: and or
  • Pipeline: |>
