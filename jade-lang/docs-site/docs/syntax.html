Syntax & grammar

Jade uses a pipe-based function syntax and familiar operators. Newlines separate statements; blocks use { }.

Functions

Declare with fn | name (params) > body. Optional return type before the pipe: fn return_type | name (params) > body.

  • Untyped params: (x), (x, y)
  • Typed params: (int | a, str | b)
fn | add_one (x) > { return x + 1 }
fn | add ( int | a, int | b ) > { return a + b }

Variables

Type-annotated: type: name = value. Examples: int: n = 0, str: s = "hi", list: xs = [1,2,3], dict: d = {"a": 1}.

Literals

Numbers: 42, 3.14. Strings: "double". Booleans: true, false. Chars: 'x'. Lists: [1, 2, 3]. Dicts: {"k": v}. Tuples: (a, b). Ranges: 1..10, 1..<10. String interpolation: "Hello {name}" or "Sum: {total}".

Keywords (reserved)

fn class if else match case while for in break continue return yield defer try catch finally panic use import let static enum trait async await cond when unless either switch converge — and type names like int str list dict when used in declarations.
