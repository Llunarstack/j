Feedback & suggestions

Report bugs, suggest features, or ask for changes. We use GitHub Issues so everything is tracked and searchable.

Bug reports

If something doesn’t work as expected—wrong output, crash, or unclear error—open a bug report. Please include:

  • Jade version (jade --version) and OS
  • Short code sample that reproduces the issue
  • What you expected vs what actually happened

Feature requests

Ideas for new language features, built-ins, or tooling are welcome. Describe the use case and, if you can, how you’d like the syntax or API to look. Check existing issues first to avoid duplicates.
