Functions

Function declarations, lambdas, and calling conventions.

Declaration

Syntax: fn [return_type?] | name (params) > body. Body is a block { } or single expression.

fn | greet ( str | name ) > {
  return out("Hello, " + name)
}

fn | add ( int | a, int | b ) > { return a + b }

fn | one () > { return 1 }

Untyped vs typed parameters

  • Untyped: (x), (x, y) — parameter names only.
  • Typed: (int | x, str | s) — type and name separated by |.

Return type

Optional return type before the pipe: fn int | name (params) > body. Return value: return expr or return (none).

Lambdas

fn | x, y > x + y
fn | (a, b) > a * b
fn | x > { return out(x) }

Arrow: > for block, => for expression-style in some contexts.

Calling

name(args), name(). Method call: obj.method(args). Pipeline: x |> fn passes x as first argument to fn.

Decorators

@decorator or @decorator(args) above a function. Built-in decorators: @memo / @cache, @timer, @log_call, @once, @throttle(sec), @debounce(sec), @profile, @trace, @deprecated("msg"), @tco (tail-call hint).
